# Background

Saints Row 2 has been outed as the worst PC port of the era *(albeit I personally disagree there is worse)*, from issues such cutscene objects de-synchronization,constant random crashing and more than sub-par performance, all of these issues have been attributed to the company who ported the release to PC, "CD Projekt Localisation Centre", GOG who was owned at the time by CD Projekt comes out of the blue in 2015 and re-releases the game on their storefront despite no promises of enhancements being done the release has edits applied to it



# So what was done?

Barring the obvious aside such as the removal of the Steam DRM making the game DRM-less, they seem to also have shipped all languages with the game rather than locked behind an option to download like in Steam, Saints Row 2 on PC picks your PC's locale as the language, otherwise the game also supports the CLIs forcelang_la with la being your language of choice.

## Changes done to the executable

The game is still the same latest patch 1.2 release as seen on Steam even barring the 49904C37 Unix timestamp in it's PE header (Mon Feb 09 15:31:03 2009)

### LARGEADDRESSAWARE enabled

GOG has patched the executable's PE to enable the LARGEADDRESSAWARE flag, which for 32 bit programs allows the application to use more than 2GB of RAM (4GB on Windows 7+ 3GB on Windows XP), this change alone significantly improves the game's stability despite it not actually using dynamic memory that much (the game during it's initialization pre allocates specific memory sizes into what they called internally as mempools  and the games logic requested memory from said mempools) it is used nonetheless in CD Projekt Localisation Centre's rendering implementation and even their leftover debug metric system! (which is always tracking in retail builds and isn't stripped), although this alone doesn't fix many other crashing instances but this one alone was the most super common one to occur.



### A sub-par frame-rate limiter

Now this one is rather peculiar, the game already comes with it's own sub-par framerate limiter but GOG has implemented what is going be the 3rd frame limiter

CD Projekt Localisation Centre's rendering implementation makes use of a command buffer to process DirectX9 calls, called in main game's thread then processed on a separate rendering thread when main game thread has finished processing GOG edits are purely assembly edits to the executable.

They've injected this frame limiter into what would process the IDirect3DDevice9::Present function

![image.png](/assets/img/posts/image.png)

```cpp
    `fps_limit_loop_counter = 0x1000000;`

    `do`

      `--fps_limit_loop_counter;`

    `while ( fps_limit_loop_counter );`
```

their implementation pretty much consists of loading the value 16,777,216 (0x1000000) into register ecx, and using the assembly instruction loop to pretty much subtract -1 from it and do nothing until it's the value 0

Doing so average out the game's framerate to ~42 FPS on my machine, but it is kind of dynamic sometimes climbing up to 46.

Now what their intention was with this largely remains unknown, a 30 FPS lock? slowing down the render thread?

What's extremely odd is the game has a framerate limiter that shipped exactly below this functionality they've added! (on my machine it limits to around 67)

### A third one for NVIDIA users

GOG also seems to ship a compatibility profile to limit the game's framerate to 30 FPS? I do not have a NVIDIA setup to confirm this. 



### Many other issues are left untouched.

Many PC port exclusive issues are left unscathed, such as the speedup/slowdown up *(despite what's commonly told it is not caused by the game being hard coded to Xbox 360's CPU clock speed, I recommend reading and watching Scanti's [post](https://www.saintsrowmods.com/forum/threads/what-causes-the-speed-up-bug-and-how-to-sort-of-fix-it.20911/)/[video](https://youtu.be/NMGoj7AV6qQ) on the topic that explains what the issue well! it's also a fix that),* graphical differences from the Xbox 360 release, performance issues,no online network capability due to Gamespy's shutdown in 2014, and much much more.
